# Тест-кейсы для API: Сервис объявлений Avito QA Internship 2026

**Хост:** `https://qa-internship.avito.com`

---

## Общие предусловия для всех тест-кейсов

1. Тестовая среда доступна по адресу `https://qa-internship.avito.com`.
2. Для выполнения запросов используется HTTP-клиент (Postman, curl или автотест на Python/pytest).
3. Заголовок `Content-Type: application/json` передаётся в каждом POST-запросе.
4. Заголовок `Accept: application/json` передаётся в каждом GET-запросе.
5. В качестве `sellerID` для тестов используются значения в диапазоне **111111–999999**, чтобы избежать коллизий с другими тестировщиками.
6. Каждый тест, требующий наличия объявления, предварительно его создаёт через `POST /api/1/item`.

# Тест-сьют 1: «Создать объявление» — `POST /api/1/item`

---

#### Предусловия для всех тестов:
С помощью вспомогательных функций должны быть сгенерированы уникальные и извлечены уникальные sellerID

## 1. Позитивные сценарии

---

### Тест-кейс 1.1: Создание объявления с валидными данными (все поля)

#### Цель:
Убедиться, что объявление успешно создаётся при передаче всех обязательных и опциональных полей с корректными значениями.

#### Предусловия:
С помощью вспомогательных функций должны быть сгенерированы уникальные и извлечены уникальные sellerID

#### Шаги:
1. Для проверки полей используем GET /api/1/item/:id

2. Отправить POST /api/1/item с корректным телом:

```json
{
  "sellerID":  get_unique_seller_id(),
  "name": "Тестовое объявление",
  "price": 1000,
  "statistics": {
    "likes": 5,
    "viewCount": 10,
    "contacts": 3
  }
}
```
3. Проверить HTTP-код ответа.
4. Проверить структуру тела ответа.
5. Проверить, что в ответе присутствует поле `id` с непустым UUID-значением.
6. Проверить, что поля `name`, `price`, `sellerID`, `statistics` соответствуют переданным значениям.

#### Ожидаемый результат:
- HTTP-статус: **200 OK**.
- Тело ответа содержит поля: `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.
- `id` — непустая строка в формате UUID.
- Значения полей соответствуют переданным в запросе.
